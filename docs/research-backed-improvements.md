# Research-Backed Improvements for Self-Harness

## Papers analyzed

| Paper | arXiv | Key idea |
|-------|-------|----------|
| **RHO: Retrospective Harness Optimization** | 2606.05922 | Self-supervised harness optimization using past trajectories: coreset selection, re-solve in parallel, self-validation + self-consistency, pairwise self-preference selection |
| **Code as Agent Harness** | 2605.18747 | Survey framing code as the harness substrate; highlights regression-free improvement and evaluation beyond final task success as open challenges |
| **SEVerA: Verified Synthesis** | 2603.25111 | Formally guarded generative models wrap each LLM call in a rejection sampler with verified fallback — zero constraint violations |
| **AutoHarness** | 2603.03329 | Small model synthesizes a code harness via iterative refinement from environment feedback; code-policy outperforms larger models |
| **Decomposing LLM Self-Correction** | 2601.00828 | Self-correction decomposed into detection, localization, correction; stronger models make deeper errors resistant to self-correction |
| **CarBoN: Calibrated Best-of-N** | 2510.15674 | Calibrated test-time scaling that adapts N based on problem difficulty |
