Verified with `make check`, `make build`, and CLI smoke runs.

## Current Verified State

The project is a production-shaped Python package with strong toy-protocol
fidelity to Self-Harness:

- `src/` layout, package metadata, CI, Makefile, ruff, mypy, pytest, build.
- Typed config, exceptions, protocols, dataclasses, patch DSL.
- Repeated evaluation with aggregate pass-count validation.
- Held-in-only proposer context.
- Schema-versioned deterministic audit artifacts.
- Declared DeepAgent-like harness surfaces: tools, skills, memory sources,
  subagents, runtime policy, and instruction surfaces.
- Proposal addressability and diversity policy.

## Remaining Gap

The package is still mostly a toy because:

- audits can be written but not read back through a supported API;
- tasks can only be created in Python code;
- the only runner is a synthetic toy runner.

The paper evaluates harness variants by running tasks in fresh environments with
verifier outcomes. We should add a small local subprocess adapter that preserves
the protocol shape (fresh workdir per attempt, command trace, verifier result)
without claiming Terminal-Bench reproduction.

## Proposed P2 Slice

1. Add `Task.metadata: dict[str, Any] = field(default_factory=dict)` for
   production adapters while preserving existing constructor compatibility.
2. Add `self_harness.audit`:
   - `AuditRun` dataclass with manifest, lineage, proposal rows, evaluation
     rows, and round directories.
   - `load_audit_run(path: Path) -> AuditRun`.
   - `summarize_audit_run(path: Path) -> AuditSummary`.
   - strict schema-version checks and clear exceptions for missing/corrupt
     artifacts.
3. Add `self_harness.adapters.local_subprocess`:
   - `load_tasks_json(path: Path) -> list[Task]`.
   - `LocalSubprocessRunner` implementing `Runner`.
   - task metadata keys: `solve_command`, `verify_command`, optional
     `workspace_template`, optional `timeout_seconds`.
   - each attempt runs in a fresh temporary workspace, optionally copied from
     `workspace_template`.
   - trace records include solve/verifier commands, exit codes, stdout/stderr
     snippets, and workdir.
   - pass/fail comes from verifier exit code.
4. Add CLI commands:
   - `self-harness audit-summary PATH`.
   - optionally `self-harness local-demo TASKS_JSON --out PATH` only if small
     enough; otherwise defer CLI runner wiring.
5. Add tests:
   - audit load/summary on demo output.
   - corrupt/missing audit file raises package exception.
   - load tasks JSON.
   - local subprocess runner passes/fails verifier-backed tasks and uses fresh
     workdirs per attempt.

## Constraints

- No Docker/Terminal-Bench integration in this slice.
- No LLM proposer API in this slice.
- Keep current demo behavior unchanged.
- Keep artifacts deterministic where generated by the engine; local subprocess
  traces may include temp paths but should still be inspectable.
